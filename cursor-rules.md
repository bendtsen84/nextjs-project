# Development Rules for This Project

- All changes and features has to be written into the /prd.md (in engelsk)

- All code, comments, variables, filenames, and system messages must be written in English.
- The website is in English. Never generate code or UI text in Danish.
- The developer (me) may write instructions in Danish, but you must always output code in English.
- Explanations outside code can be in Danish, unless explicitly asked otherwise.
- Follow Next.js 15 App Router best practices.
- Use MongoDB with Mongoose, TailwindCSS, DaisyUI, Stripe
- models are placed i /models
